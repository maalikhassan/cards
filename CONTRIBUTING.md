# Contributing to LUCK & LOGIC

Thank you for your interest in improving the LUCK & LOGIC card game! This guide will help you understand how to contribute.

## How to Contribute

### Reporting Issues
Found a typo or design issue? Please open an issue with:
- A clear description of the problem
- Where it appears (which card)
- Screenshots if possible
- Suggested fix (optional)

### Submitting Card Ideas
Have ideas for new cards? Submit them with:
- **Card Name:** Clear, memorable title
- **Card Type:** Action or Challenge
- **Description:** Clear, concise rules
- **Game Balance:** How does it affect gameplay?
- **Playtesting Notes:** Any insights from testing

### Making Edits

1. **Format:** Follow the existing card structure in `cards.html`
2. **Consistency:** Use the same style and language as existing cards
3. **Balance:** Ensure new cards aren't too overpowered
4. **Testing:** Print and test your cards before submitting
5. **Documentation:** Update this file if you add significant changes

### Code Style Guidelines

**HTML/CSS:**
- Use consistent indentation (4 spaces)
- Add comments for card type sections
- Keep card styling consistent
- Test on multiple browsers

**Card Writing:**
- Use active voice ("Do X" not "You should do X")
- Keep descriptions under 2 lines when possible
- Use clear, jargon-free language
- Specify point values and conditions clearly

## Customization Ideas

### Card Balance Suggestions
- Limit overpowered cards (no more than +10 points)
- Ensure challenge cards are solvable by most players
- Balance reward/risk ratio

### Visual Improvements
- Better card icons (emoji or SVG)
- Enhanced print quality
- Dark mode/light mode options
- Multiple color themes

### Feature Additions
- Card deck statistics
- Suggested card combinations
- Beginner/Advanced card sets
- Interactive card designer
- Mobile version

## Development Setup

### Requirements
- Any modern web browser
- Text editor (VS Code recommended)
- Optional: Git for version control

### Testing Your Changes
1. Edit `cards.html` in your text editor
2. Save the file
3. Open in browser (or refresh if already open)
4. Test print preview (Ctrl+P)
5. Check mobile responsiveness

### Submitting Changes
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/card-name`
3. Make your changes
4. Test thoroughly
5. Commit with clear messages: `git commit -m "Add CARD_NAME card"`
6. Push to your fork
7. Open a Pull Request with description

## Community Guidelines

- **Respect:** Treat all contributors with respect
- **Constructive:** Provide helpful, constructive feedback
- **Inclusive:** Cards should be fun for all ages and abilities
- **Family-friendly:** Keep all content appropriate
- **Balanced:** Avoid cards that create unfair advantages

## Questions?

- Check existing documentation in README.md
- Review existing cards for examples
- Ask in issues or discussions
- Reach out to project maintainers
